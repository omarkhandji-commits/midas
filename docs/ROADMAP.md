# MIDAS Roadmap

Principle: add autonomy only when the control surface exists first.

## Current Release Target

MIDAS is now a proof-first business operator with:

- central runtime wiring;
- CLI product surface;
- local dashboard;
- memory, approvals, receipts, budget fuse, Sentinel, context economy;
- business asset generation;
- Market Radar;
- provider diagnostics for local and cloud LLMs;
- multi-LLM council command;
- user-installed schedule recipes;
- security-first skill registry;
- deterministic transparency evals.

## Release Blockers

- ShipVitals review must be recorded.
- README quickstart must pass from a clean checkout.
- Full test suite and `midas eval` must pass.
- No secret or PII fixtures.
- No outbound action without approval.
- No revenue guarantee or fake benchmark claim.

## Near-Term Improvements

- Better dashboard provider/config screens.
- Optional OCR, audio, and video adapters behind explicit extras.
- Better visual regression and accessibility coverage for the dashboard.
- Signed remote skill bundles.
- More provider live smoke tests behind opt-in flags.
- Example replay demo with screenshots and GIF.

## Later

- Consent-first phone-call adapter.
- CRM integrations with approval-gated writes.
- Multi-project portfolio memory.
- Independent third-party review before a public “ready” claim.

## Non-Goals

- No automatic spam.
- No scraping personal data for outreach.
- No self-scored revenue benchmark in V1.
- No “certified” or “compliant” claims.
