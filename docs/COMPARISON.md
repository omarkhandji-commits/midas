# Positioning

MIDAS focuses on self-hosted LLM workflows that need review before action. It is
not a replacement for every agent workspace or every coding assistant.

## Where MIDAS Fits

MIDAS is strongest when a workflow needs:

- a local dashboard and CLI;
- an approval queue before risky actions;
- signed receipts and an audit trail;
- budget controls before model calls;
- Ollama or user-supplied cloud provider keys;
- MCP-oriented tool workflows;
- proof export for later review.

## Where MIDAS Is Not The Right Tool

MIDAS is not the right choice when you need:

- a hosted SaaS account with no local setup;
- a fully automatic system that acts without review;
- a general chat assistant only;
- a claim of certification or regulatory compliance;
- guaranteed outcomes.

## How To Evaluate It

Use concrete checks:

1. Run the launcher or CLI from a clean checkout.
2. Queue an action that changes a file or calls a tool.
3. Review the approval card.
4. Approve or reject it.
5. Export the receipt proof.
6. Verify the receipt ledger with the standalone verifier.

That path shows what MIDAS actually provides: local operation, controlled
actions, and a reviewable audit trail.
